# askflow概要
## はじめに
- askflowはユーザーの思考を可視化するためのツール。
- 思考とは、質問と、それに対する回答を繰り返すプロセス。
- askflowはこのプロセスを「フロー」として捉え、ユーザーが質問と回答を記録・整理できるようにする。
- これにより、ユーザーは自分の思考過程を人に伝えたり、自分で振り返ったりできる。

## 主な特徴
- ノードベースの編集（質問ノード、回答ノード、作業ノードの3種類）
- ノード同士を矢印で接続してフローを表現
- ノード内は Markdown によるリッチテキスト対応
- ローカルファーストの保存方針（ユーザーデータはローカルに保存し、エクスポート可能）

## 用語定義
- 質問ノード: ユーザーが問いを記述するノード
- 回答ノード: 質問に対する応答・考察を記述するノード
- 作業ノード: 実行すべきタスクや検証内容など、アクションを表すノード

## データ方針
- デフォルトはローカル保存（ユーザーがデータを完全にコントロール可能）
- Markdown / JSON など一般的なフォーマットでのエクスポート機能を提供
- クラウド同期などは現段階では提供しない。 

## 設計原則
- シンプルさ: 最小の操作でフローが作れること。
