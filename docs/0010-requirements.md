# 001 - 要件（個人開発向け・簡潔）

## 概要
- プロジェクト: askflow
- 目的: 「質問→回答」のフローを手軽に記録・振り返る個人用ツール
- スコープ: 当面は単一開発者による Web MVP（ローカル保存優先）

## MVP（最小限）
- 新規フロー作成（質問/回答/作業ノード）
- ノードの編集（Markdown 対応、保存/読み込み）
- ローカル保存とエクスポート（Markdown/JSONといった一般的な形式に対応）
- シンプルな検索（キーワード／タグ）

## 非機能（簡潔）
- クリーンな設計: 保守性・拡張性を意識したシンプルな構造
- セキュリティ: 初期はローカル保存で運用（クラウド同期は実装しない）
- パフォーマンス: 日常使用で違和感がないこと（目安: 軽快に動くこと）

## 制約・前提（個人開発）
- ドキュメントは最小限に留める
- 企業向けの大規模要件やコンプライアンス対応は今は考慮しない（必要なら個別対応）

## 短期マイルストーン（例）
- 0: 技術スタックを決める
- 1: リポジトリ初期構成、簡易 UI スケルトン
- 2: 基本保存・編集・検索を実装して手元で動かす

## 開発時の意思決定
- プラットフォーム: デスクトップで動作するローカルアプリケーション
- 認証: 初期フェーズでは認証は不要（ローカルのみで運用）

## 受け入れ（簡易）
- 自分が 5 分以内にフローを作れること
- 保存・読み込みが手元で問題なく行えること
