# 0020 - 技術スタック（簡潔）

## 前提
- ターゲット: デスクトップで動作するローカルアプリケーション（個人開発）
- 方針: ローカル保存を最優先に、早く反復できるスタックを選ぶ

## 推奨スタック（第一候補）
- フロントエンド: TypeScript + React
- デスクトップシェル: Electron (Chromium + Node を同梱、学習素材とエコシステムが豊富)
- ビルド/開発ツール: Vite（迅速な HMR と小さな設定）
- ストレージ: ローカルファイル（ユーザーのアプリデータ領域に JSON または SQLite）
- テスト: Vitest（ユニット） + Testing Library（UI）
- リント/フォーマット: ESLint + Prettier

## 代替案
- Tauri + React
  - 長所: バイナリが小さく、起動が速い。セキュリティ面で有利な設計ができる。
  - 短所: Rust ツールチェインが必要で、ネイティブ連携の実装に多少の学習コストがある。

## 選定理由（短く）
- TypeScript + React は個人開発で生産性が高く保守もしやすい。
- Electron は学習リソースや既存サンプルが豊富で、王道的なデスクトップ開発を学ぶのに向く。Node エコシステムがそのまま使えるため開発速度が速い。
- ローカルファイル（最初は JSON）で始め、必要に応じて SQLite に移行する方針がシンプルで実装が早い。

## 短期イテレーション案（次の 3 ステップ）
1. Electron + Vite + React の最小スケルトンを作る（Hello world ウィンドウ、保存先の確認）
2. ノードモデル（質問/回答/作業）を JS オブジェクトとして保存/読み込みする実装
3. シンプルな検索（キーワード）を実装して UX を確認する

---

（補足）Electron を採用する場合のテンプレートや、Tauri のセットアップ手順を用意できます。進めたい方向を教えてください。
